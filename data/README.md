# data/

Datos estructurados del sitio, separados de la capa de presentación.

## Archivos

- `services.ts`: catálogo de servicios y capacidades.
- `projects.ts`: proyectos del portafolio y estado de avance.
- `process.ts`: pasos del proceso de trabajo.

## Regla de mantenimiento

Actualizar primero el archivo de datos y luego ajustar el renderizado en componentes/páginas si hace falta.
